# 検出の意味論と合否基準 — 真の貫通・偽陽性・可視合否

このドキュメントは**検討資料**です。貫通「検出」が拾うものと、修正の「目的」のあいだにあるギャップ — すなわち **何を直すべき貫通とみなし、何を合格とするか** — を整理します。検出ロジックそのものは [DESIGN.md](DESIGN.md) §6、サポート範囲の緑/黄/赤判定は §9、保証層を過信しない側(GREEN を信じすぎない)の議論は [DIAGNOSTIC_HONESTY.md](DIAGNOSTIC_HONESTY.md) を参照。本書はその対 — **検出層が過剰報告する側**を扱う。最終更新: 2026-06-17。

## 0. 要旨

検出器が測るのは**幾何的な貫通**(衣装頂点が素体表面より内側にあるか)。だが本ツールの目的は**可視面の正しさ**(観察者から見て素体が衣装を突き破って見えないこと)。この2つは一致しないことがある:

- **真の貫通** — 可視面で素体が露出する。直すべき対象
- **偽陽性** — 検出は内側と判定するが、可視面に破綻はない/そもそも幾何が縮退している。直す必要がない、あるいは直してはいけない

合否は **残留検出数のゼロではなく、可視の外側面に素体が出ていないか**で見る。残留0は厚物衣装などでは達成不能かつ不要。

## 1. 検出が測るもの vs 目的

検出器([DESIGN.md](DESIGN.md) §6)は、衣装の各頂点について素体コライダの符号付き距離を見て、表面より内側(負側・マージン込み)にあるかを判定する。これは**幾何的な貫通の代理指標**であって、目的そのものではない。

本ツールの目的は、観察者から見て**素体が衣装の可視面を突き破って見えないこと**。可視面とは衣装の最も外側のメッシュ(シルエットを成す面)。検出が拾う「内側にある頂点」のうち:

- **可視面の頂点が内側** → 素体が露出 = **真の貫通**(直す)
- **不可視の頂点(内側壁・畳まれた頂点)が内側** → 可視面は無傷 = **偽陽性**(直さない/直せない)

検出器は単独ではこの2つを区別しない(可視性・遮蔽を見ていない)。区別は合否基準(§2)と偽陽性の類型(§3)で補う。

## 2. 合否基準:可視の外側面

| 優先 | 何を見るか | 意味 |
|---|---|---|
| **一次** | 可視の外側面から素体が露出していないか(目視。肌見せ境界×関節を重点) | ツールの目的そのもの。手順は [E2E_TEST_GUIDE.md](E2E_TEST_GUIDE.md) |
| **二次** | Preflight の残留貫通が下がること | 押し出しが届いた計算上の指標。**ゼロは必要条件でない**(§3.3 の偽陽性が残る) |

- 残留が0でなくても、**可視面がクリアなら合格**(残留は偽陽性ぶん)
- 残留が0でも、**可視で破綻(二重適用・伸び・膨らみ)があれば不合格**
- つまり残留数と可視合否は直結しない。残留は「到達度」、合否は「可視面」

## 3. 偽陽性のカタログ

検出が拾うが「直すべき真の貫通」ではないものを類型化する。対処はいずれも **押し出しを強める方向ではなく、偽陽性と認識して検出から除外する/原因を名指しする**方向。

### 3.1 プロキシ形状の近似誤差(解決済)

円断面カプセルは胴・足の楕円/扁平断面を表せず、ヌルテスト(対応素体)でも内側を素体と誤判定して偽貫通を出した。→ **メッシュSDF コライダ**([DESIGN.md](DESIGN.md) §6)で素体の実形状を使い解消。**プロキシの形状忠実度を上げれば消える**型の偽陽性。

### 3.2 縮め/隠しシェイプキー(フェーズ3課題)

衣装の一部を意図的に縮める/隠す独立シェイプキー(Blendshape Sync 無し)は、畳んだ頂点を体内深部へ押し込む。検出は深い貫通として正しく拾うが、これは**畳まれて不可視**な頂点で、直すべき可視貫通ではない。偽RED→当該レンダラーごと skip になり、同レンダラーの実貫通も巻き添えになりうる。→ ROADMAP フェーズ3「縮め/隠しシェイプキーへの堅牢性と原因の名指し」。畳み込み(縮退三角形・体内密集)を検出から除外し、原因を名指しする。

### 3.3 厚物/二重メッシュ衣装の内側壁(新規)

靴・ブーツ・厚手アウター等、**内側メッシュと外側メッシュを持つ厚みのある衣装**では、内側壁が素体表面より内側に正当に位置する(足は靴の内壁のさらに内側にある)。検出器はこの内側壁頂点を貫通として拾うが、**外側(可視)メッシュには素体が一切届かず、可視面はクリア**。

- これは押し出しで解くべき対象ではない(内側壁を押し出すと衣装が歪む)。**可視外側面がクリアなら合格**
- 厚物を含む実衣装では Preflight 残留は0にならない(内側壁ぶんが残る)。残留数が**可視の合否と直結しない**ことの典型例
- 実機観察(2026-06-17): 同一トポロジ姉妹アバターのスケール差クロステスト(Bake 後)で、薄物(パンツ)の体型差貫通は解消、厚物(靴)に数百頂点の残留が出たが、可視外側面に素体露出はなくクリア。残留の大半はこの内側壁

## 4. 含意

- **残留メトリクスの読み方** — Preflight の `remainingPenetrating` は「押し出しの到達度」の指標であって合否そのものではない。厚物衣装では下限が0でない。合否は可視面で取る([E2E_TEST_GUIDE.md](E2E_TEST_GUIDE.md) §8)
- **ソルバ比較への影響** — 残留数で2ソルバを比べる際、差が偽陽性領域(内側壁等)に乗っていると**可視品質の差を反映しない**。比較は偽陽性を含まないケースか、可視面で行う([DEFORMATION_METHODS.md](DEFORMATION_METHODS.md) §3.1)
- **フェーズ3の堅牢化** — §3.2・§3.3 の偽陽性は「検出から除外し原因を名指しする」方向で扱う(ROADMAP フェーズ3)
- **エコシステム標準への含意** — 可視合否という基準は、診断を互換性標準へ育てる際の「何を保証するか」の定義にも関わる(保証側の議論は [DIAGNOSTIC_HONESTY.md](DIAGNOSTIC_HONESTY.md))

---

*検出は代理指標、目的は可視面。両者のギャップを偽陽性として明示的に管理することが、押し出しの過剰適用(衣装の歪み)を防ぐ。*
